# MMORPG Board — доска объявлений для MMORPG-игроков

## Описание проекта

MMORPG Board — это веб-приложение, разработанное на Django, предназначенное для размещения и поиска игровых объявлений. Пользователи могут публиковать объявления различных категорий, оставлять отклики на объявления других пользователей, а также управлять собственными объявлениями и откликами.

Проект реализован с использованием Django CBV (Class Based Views), Django Allauth для аутентификации и подтверждения электронной почты, Django ORM для работы с базой данных и Celery для выполнения фоновых задач.

---

## Основной функционал

### Регистрация и авторизация

* Регистрация новых пользователей.
* Авторизация пользователей.
* Подтверждение электронной почты через код, отправляемый на email.
* Пользовательский профиль с дополнительными полями:

  * фотография;
  * дата рождения;
  * информация о себе.

### Работа с объявлениями

Авторизованный пользователь может:

* создавать объявления;
* редактировать собственные объявления;
* удалять собственные объявления;
* просматривать список своих объявлений;
* просматривать объявления других пользователей.

Каждое объявление содержит:

* заголовок;
* текст объявления;
* категорию;
* автора;
* дату публикации.

### Работа с откликами

Авторизованный пользователь может:

* оставлять отклики на объявления других пользователей;
* просматривать собственные отклики;
* просматривать отклики, оставленные на его объявления.

Для каждого отклика предусмотрены статусы:

* ожидает ответа;
* принят;
* отклонён.
* Если пользователь зарегистрировался, но не подтвердил свою почту через код, то пользователь может только оставлять отклики, создавать объяление не сможет

Автор объявления может принять или отклонить поступивший отклик.

### Почтовые уведомления

Система автоматически отправляет электронные письма:

* при регистрации пользователя;
* при подтверждении электронной почты;
* при поступлении нового отклика на объявление;
* при принятии отклика;
* при отклонении отклика.

### Еженедельная рассылка

С помощью Celery и Redis реализована фоновая задача, которая формирует еженедельную рассылку пользователям с подтверждённой электронной почтой.

В рассылку включаются:

* новые объявления за последние 7 дней;
* количество новых объявлений;
* список последних опубликованных объявлений.

## Логика работы системы

### Пользователь

1. Пользователь проходит регистрацию.
2. На электронную почту отправляется код подтверждения.
3. После успешного подтверждения почты пользователь получает доступ ко всем функциям системы.

### Объявления

1. Пользователь создаёт объявление.
2. Объявление отображается на главной странице.
3. Другие пользователи могут просматривать объявление и оставлять отклики.

### Отклики

1. Пользователь оставляет отклик на объявление.
2. Автор объявления получает уведомление по электронной почте.
3. Автор объявления принимает или отклоняет отклик.
4. Автор отклика получает уведомление о результате рассмотрения.

### Безопасность

Реализованы ограничения доступа:

* неавторизованные пользователи не могут создавать объявления и отклики;
* редактирование и удаление доступны только автору объекта;
* отдельные действия защищены проверкой прав доступа через UserPassesTestMixin и проверки владельца объекта;
* используется CSRF-защита Django.
